\section{Introduction}
Ideally, phylogenetic inference of whole-genome analysis reveals that the entire genomes evolved under a single evolutionary history. Reality, however, is more complex. For example, if two related viruses infect a single cell, during genome amplification RNA-dependent RNA polymerase (RdRp) can change the template from one viral genome to the other, commonly referred to as template switching \citep{simon2011rna}. This process links parts of the co-infecting genomes, generating a hybrid virus variant. The different regions of this recombinant genome now reflect the different evolutionary histories of their origin. Similar effects occur via lateral gene transfer between bacteria or crossover during meiosis in eukaryotes \citep{Ochman2000, Hunter2015}.

Recombination between highly divergent sequences may disrupt essential gene functions, so successful recombinants typically arise from homologous regions where sequence and gene order are conserved.
Consequently, recombinant genomes usually retain enough similarity to be captured in a single multiple sequence alignment (MSA). However, treating such an alignment as a single evolutionary unit can obscure conflicting phylogenetic signals \citep{Posada2002}. Sliding-window approaches address this problem by reconstructing a phylogenetic tree for each successive, overlapping window along the alignment \citep{Salminen1995, lole1999fulllength-6e0} (Fig.~\ref{fig:overview}a,b). Because each window samples only a short genomic segment of the alignment, its tree reflects the local evolutionary history of that region alone. Where no recombination has occurred, consecutive windows yield similar or identical topologies (i.e., branching structures independent of branch lengths); where recombination has occurred, the topology shifts to reflect the distinct ancestry of the recombinant segment, and the genomic position at which this shift occurs approximates the location of a recombination breakpoint.

As the window progresses, the tree topologies change to reflect the local evolutionary history, producing a sequence of trees ordered by genomic position. Topological shifts in this sequence mark candidate recombination breakpoints. This approach requires sufficient sequence divergence: recombination between nearly identical parents leaves the tree structure unchanged and thus escapes detection.

Several tools exist to analyse recombinant alignments. RDP5, SimPlot, SimPlot++, and TOPALi focus primarily on detecting recombinants and locating breakpoints \citep{martin2020, lole1999fulllength-6e0, samson2022simplot-617, Milne2004}. Tree House Explorer takes a complementary approach, showing the distribution of tree topologies across genomic windows in a genome browser \citep{Harris2022THEx}. However, it does not visualise how trees from consecutive windows relate to each other, leaving users unable to identify \textcolor{red}{which subtrees rearrange or how their source and target parent branches differ}.

The scale of modern datasets compounds this challenge. Small \textcolor{red}{sliding-window} step sizes generate hundreds or thousands of largely redundant trees that must be inspected for subtle rearrangements, whereas larger step sizes can produce abrupt topological changes between consecutive windows, making it difficult to track \textcolor{red}{which subtrees change parent branch}. For example, sliding-window analysis of a 10 \,kb HIV genome with a 10 \,nt step produces approximately 1,000 trees \citep{abecasis2007}. Identifying the windows where tree rearrangements occur requires tracing branching structures across hundreds of trees, a task poorly suited to static visualisation \citep{2009_Graham_Kennedy}.

To address these challenges, we developed Phylo-Movies, \textcolor{red}{which} animates topological rearrangements between consecutive phylogenetic trees. \rev{R3c3}{Users may supply a pre-computed tree sequence inferred with their preferred software, or upload an MSA for rapid exploratory tree generation using IQ-TREE~3 \citep{Wong2026IQTREE3} or FastTree \citep{Price2010} with user-defined window and step sizes. For publication-grade analyses, users can import ordered tree files inferred externally with IQ-TREE, RAxML, or another preferred phylogenetic program.} Trees can be rooted using either an outgroup or midpoint rooting. \textcolor{red}{Phylo-Movies visualises topological changes by representing the transition between two consecutive trees as a series of rooted subtree prune-and-regraft (SPR) moves. An SPR move prunes a subtree from one location in a rooted tree and regrafts it onto another branch, yielding a new topology after a single rearrangement \citep{BordewichSemple2005}. Each SPR move is animated with the affected subtree highlighted, allowing regions of substantial topological change within the tree sequence to be readily identified.} We next describe the algorithmic framework underlying the animation and present two use cases illustrating its analytical value.

\subsection{How to Animate Topological Changes Between Trees}
\label{howTo}

Phylo-Movies highlights and animates topological changes within a sequence of phylogenetic trees. Visualisation requires rooted trees, which can be obtained using either an outgroup or midpoint rooting \citep{edwards2019}. Trees are displayed using a circular layout \citep[cf.][chap.~36]{felsenstein2004}. \rev{R2c4}{This layout was chosen because it keeps moving subtrees visually trackable during animation, although rectangular animated layouts are an important future extension for users who prefer conventional phylogram-style inspection of internal structure.}

To visualise the whole sequence of trees, topological changes are animated \rev{R3c4}{between each pair of consecutive trees, transforming the (preceding) source tree into the (succeeding) target tree} (cf. Fig.~\ref{fig:overview}b). To define this transformation, we use the concepts of splits and consensus trees \citep[cf.][]{bryant1997}.

Every internal branch defines a (non-trivial) split that partitions the leaves into two subsets. Comparing the source and target trees identifies three categories of splits: splits shared by both, splits unique to the source tree, and splits unique to the target tree. The Robinson–Foulds distance (RF) between the two trees is defined as the number of splits unique to the source plus the number unique to the target \citep{ROBINSON1981131, RobinsonFoulds2006}.

To visualise the transition from the source tree to the target tree, Phylo-Movies uses the strict consensus tree, which contains only the splits present in both trees, with branch lengths set to the mean of the corresponding values in the source and target trees. The transformation is organised around a \textbf{\textcolor{red}{pivot edge}}, defined as a branch shared by both trees whose immediate descendant branches differ between the source tree and the target tree.

This process is shown in the workflow in Fig.~\ref{fig:overview}. From the alignment in Fig.~\ref{fig:overview}a, window subalignments are extracted to reconstruct the trees in Fig.~\ref{fig:overview}b. Fig.~\ref{fig:overview}c shows a topological rearrangement in the upper part of the trees, namely the \textcolor{red}{SPR move of} \textbf{leaf~3} (marked magenta). In the source tree (1, Fig.~\ref{fig:overview}c), leaf~3 is sister to leaf~30. To reach the target tree topology (5, Fig.~\ref{fig:overview}c), \textcolor{red}{where leaf~3 has the branch subtending leaves 30, 40, 41, 50, and 51 as its parent branch}, the algorithm identifies the \textbf{\textcolor{red}{pivot edge}} (thick blue arc) and performs the rearrangement relative to it.

The transformation begins by shrinking to zero the branch leading to the subtree of leaves~3 and 30, as well as the branch rooting the subtree containing 3, 30, 40, and 41 (Fig.~\ref{fig:overview}c. 1). Both branches correspond to splits unique to the source tree and remain visible in tree~2) as black dots. These zero-length branches are then removed, yielding the intermediate strict consensus tree (Fig.~\ref{fig:overview}c.1). Subsequently, leaf~3 moves to its correct position within the multifurcation (Fig.~\ref{fig:overview}c.4), and finally the two new branches unique to the target tree are inserted and expanded in Fig.~\ref{fig:overview}c.5: the branch rooting subtree (30, 40, 41) and the branch rooting subtree (30, 40, 41, 50, 51).

Fig.~\ref{fig:overview}d presents a more complex scenario where two taxa (\textbf{leaf~1} and \textbf{leaf~2}, both marked magenta) \textcolor{red}{change parent branch}. In the source tree 1), leaf~1 \textcolor{red}{forms a subtree with} leaves 10 and 11, while leaf~2 \textcolor{red}{forms a subtree with} leaves 20 and 21. To transform towards the target tree 5), the branches rooting the subtrees (1, 10, 11) and (2, 20, 21) are shrunk to zero length, see black nodes at the end of the magenta branches in tree 2). In the next step (tree 3), these zero-length branches are removed yielding the intermediate strict consensus in tree 3) where both leaves share the same parent multifurcation outside the \textcolor{red}{pivot edge}. Again the branches are reordered to match the target topology (while the consensus topology remains static - see tree 3). Then, the new branch rooting subtree (1, 20, 21) is inflated (tree 4) and, finally, the branch for subtree (2, 10, 11) is inserted in tree 5. This demonstrates how multi-taxon rearrangements are decomposed into sequences of branch collapse and expansion.

\rev{R3c4}{For each pair of consecutive trees in the input sequence, Phylo-Movies compares the source tree with the target tree. The shared pivot edge isolates the local conflict between source-unique and target-unique splits, and the affected subtrees are then animated as rooted SPR moves \citep{BordewichSemple2005}. Subtrees that remain structurally identical between the source and target trees are retained, and only those whose placement differs are animated.}

Once the \textcolor{red}{moving subtrees} are identified, their relocation to the target position is animated while the consensus structure remains static. To maintain visual clarity, Phylo-Movies animates these \textcolor{red}{SPR moves} sequentially. When a single \textcolor{red}{moving subtree} contains multiple internal conflicts, a heuristic decomposes the \textcolor{red}{rearrangement} into smaller sequential transitions.

For each rearrangement, the visualisation animates the corresponding SPR move by smoothly transitioning from the source tree layout to the target tree layout, producing a sequence of frames (default: 30). To minimise extraneous motion, the tool optimises the leaf ordering of the target tree to match the source layout, ensuring that only subtrees involved in topological changes appear to move. Additional navigation and analysis features are described below.

The Phylo-Movies user interface (Fig.~\ref{fig:phylo_movies_norovirus}) consists of several synchronised panels. The \textit{animation and tree viewer panel} (Fig.~\ref{fig:phylo_movies_norovirus}A) displays the transformation between trees, where \textcolor{red}{moving subtrees} are highlighted.

The \textit{playback control panel} (Fig.~\ref{fig:phylo_movies_norovirus}C) includes buttons to play, stop, and step through the animation forward and backward. The \textit{timeline chart} below (Fig.~\ref{fig:phylo_movies_norovirus}C) tracks the genomic location with a playhead. Clicking or dragging the playhead adjusts the animation position. 

The distance plot below the timeline shows a normalised Robinson--Foulds series \citep[RF;][]{ROBINSON1981131,RobinsonFoulds2006} between adjacent input trees. Following the standard split-based definition, each internal branch defines a non-trivial bipartition of the taxa, and RF is the number of bipartitions present in one tree but not the other. The plotted value normalises this count by the total number of internal bipartitions in both trees, so values range from 0 to 1, with branch lengths ignored.

The weighted RF series is not normalised: it is the raw sum, over all rooted weighted splits that appear in either tree, of the absolute difference between the two split lengths (using 0 for splits that are absent in one of the trees). Users can switch between the normalised topology series, the raw weighted series, and the raw tree-size scale.

The \textit{alignment panel} (Fig.~\ref{fig:phylo_movies_norovirus}B) displays the genomic alignment synchronised with the current tree, allowing users to associate genome positions with topological changes. A \textit{setup panel} on the left (Fig.~\ref{fig:phylo_movies_norovirus}D) provides configuration options for the visualisation and toggles synchronisation between the alignment viewer and the tree state. A \textit{control panel} on the right enables colouring of subtrees and taxa by metadata annotations and metadata-defined groups.

Phylo-Movies also supports side-by-side tree comparison: users can pin any tree from the sequence as a reference (Fig.~\ref{fig:phylo_movies_norovirus}A, right), helping to track where subtrees were positioned before and \textcolor{red}{how their parent branches differ} relative to a chosen starting point. A comparative mode links corresponding taxa between phylogenies to highlight structural differences. Rendering is implemented via WebGL to support large trees and alignments at sufficient speed.

The visualisation supports several customisation options designed for scientific interpretability. Subtree colouring propagates a leaf colour to internal branches only when all descendants share that colour, \textcolor{red}{making metadata-homogeneous descendant sets visually distinct}. Users can load metadata from a CSV file to define metadata-defined groups (e.g., genotype, host, or geographic origin) and interactively switch between grouping schemes. \textcolor{red}{Phylo-Movies also reads and displays values associated with internal nodes or ancestral branches, such as IQ-TREE SH-aLRT branch-support values or bootstrap split-frequency support values, so that users can assess whether an animated movement involves well-supported clades or mainly low-support, uncertain placements.} Node size can be scaled, and highlighted elements—pivot edges, marked subtrees, or structures from previous frames—are rendered with increased radius or thicker outlines to improve perceptual salience. Edge highlighting employs explicit visual precedence, ensuring that marked subtrees and active change edges remain distinguishable from baseline colouring.

\rev{R2c3,R2c10}{The tree viewer provides zoom and pan interaction, together with controls for font size, branch width, node size, and label spacing. Branch lengths can be displayed in a metric mode, in which rendered lengths remain proportional to the input Newick branch lengths, or in visual modes designed for readability during animation. The default animation mode applies a square-root transform followed by root-to-tip normalisation within each tree, stabilising the apparent tree size across frames; this mode is intended for visual tracking of topology and should not be interpreted as preserving absolute evolutionary scale. Absolute branch-length changes remain available through the scale-tracking panel and branch-length-aware distance summaries.}

\section{Use Cases and Results}
\subsection{Recombination-Induced Topological Shifts in Norovirus}
Recombination drives norovirus evolution by enabling viruses to acquire novel capsid variants while retaining a functional polymerase, thereby evading host immunity \citep{Chhabra2019, Lindesmith2008, Bull2005}. Because the polymerase gene (ORF1) and capsid gene (ORF2) frequently have distinct evolutionary origins, a dual-genotype nomenclature is used; \textcolor{red}{for genogroup II viruses this is commonly written as GII.\textit{y}[P\textit{x}]}, where GII.\textit{y} denotes the capsid genotype and P\textit{x} the polymerase genotype \citep{Chhabra2019}. Matching genotypes (e.g., GII.4[P4]) suggest a shared evolutionary history, whereas mismatched combinations (e.g., GII.4[P16]) are consistent with recombinant origins.

To investigate how tree topology shifts across the recombination breakpoint, we used Phylo-Movies to visualise a sliding-window phylogenetic analysis on a global dataset of 334 full-genome norovirus sequences (8,058~bp alignment) from 32 countries, sampled between 1968 and 2025. Sequences were downloaded from the Nextstrain-curated GenBank dataset \citep{Hadfield2018}, subsampled using Augur \citep{Huddleston2021}, and genotyped using Nextclade \citep{Aksamentov2021}. \textcolor{red}{Using capsid labels from VP1/ORF2 annotations and polymerase labels reduced to numeric P-types, the dataset contains 30 capsid labels and 28 polymerase P-types. Among the 302 sequences with comparable numeric capsid and polymerase assignments, 167 had discordant genotype numbers (55.3\%; 50.0\% of all 334 sequences).}

The sequences were aligned using MAFFT v7.526 \citep{Katoh2013} and trimmed using trimAl \citep{CapellaGutierrez2009}. \textcolor{red}{For the promoted Norovirus SH-aLRT branch-support example, trees were inferred using IQ-TREE~3 \citep{Wong2026IQTREE3} in fast-search mode under the GTR+G model. The analysis used 17 centre-based sliding windows of 1{,}000~bp stepped every 500~bp, with boundary windows clipped to the alignment ends, and 1{,}000 SH-aLRT replicates per window to annotate internal branches with branch-support values. Here, SH-aLRT branch support refers to the support percentage reported for an internal branch, i.e. a local measure of support for the displayed split or placement within a window, not a taxon-level recurrence measure or a probability of recombination.}
\textcolor{red}{The resulting midpoint-rooted tree sequence was visualised with Phylo-Movies, and Robinson--Foulds distances between adjacent windows were used together with the SH-aLRT branch-support labels to identify regions of pronounced topological change and inspect the local support of the displayed placements} (Fig.~\ref{fig:phylo_movies_norovirus}).

\textcolor{red}{At the ORF1/ORF2 junction (approximately position 5,100~bp), the largest normalised Robinson--Foulds distance occurred between the windows centred at 5{,}001 and 5{,}501~bp (RF = 0.565). Weighted Robinson--Foulds distances were also elevated across the following ORF2-side transitions, reaching their maximum between the windows centred at 6{,}001 and 6{,}501~bp (weighted RF = 21.16). The RF-peak windows had median SH-aLRT branch-support values of 88.0 and 85.8, respectively.} For the highlighted recombinant groups, ORF1 windows show polymerase-genotype-associated placements, whereas ORF2 windows show capsid-genotype-associated placements.

\textcolor{red}{The clearest group-level shifts were observed for P16 recombinant lineages. GII.4[P16] recombinants were placed near other P16 sequences in ORF1 but near GII.4 sequences after the transition into ORF2. The GII.2[P16] group (13 sequences) showed a related but more mixed transition from the P16-associated tree region in ORF1 towards GII.2-associated placements in ORF2. GII.6[P7] variants (14 sequences), sampled from eight countries across the Americas, Asia, and Africa, were also among the frequent moving-subtree genotype labels, but their P7-associated placements were less cleanly resolved than the P16 examples and are therefore treated as exploratory.} These patterns illustrate the mosaic origin of recombinant genomes: segments derived from different \textcolor{red}{parental genomic backgrounds} form contiguous blocks that associate with distinct \textcolor{red}{taxon sets} depending on genomic position. Because the animation decomposes topological transitions into sequential SPR moves, the relocation of these subtrees can be followed step by step. This approach can also be applied to sequences of unknown origin: by including them in the analysis, one can observe which \textcolor{red}{genotype-associated tree regions} they join in different genomic regions, thereby suggesting their polymerase and capsid genotype assignments.

A feature of the alignment viewer allows sequences to be ordered according to the current tree layout while colour-coding nucleotide columns, enabling direct visual identification of matching and mismatching regions within the alignment. When the data are noisy, numerous small subtree rearrangements may occur and can be tedious to follow individually. \textcolor{red}{The distance plot highlights windows with larger topological changes, while the SH-aLRT branch-support labels help separate well-supported genotype-associated placements from local low-support rearrangements. Across the full tree sequence, 1{,}616 SPR moves were recorded. Across the 17 input trees, 5{,}322 internal SH-aLRT branch-support labels were available; the median SH-aLRT score was 86.0 across all labels and 91.7 when zero-support labels were excluded, with 3{,}713 labels (69.8\%) at or above 70. When SPR events were summarised by whether a genotype label was present in the moving subtree, the most frequent labels were GII.17[P17], GII.4[P4], GII.4[P31], GII.4[P16], GII.6[P7], and GII.2[P16].}

\textcolor{red}{The complete SH-aLRT branch-support tree series, window table, and example configuration are provided with the example datasets and are publicly available in the project’s GitHub repository. The generated browser example contains the temporal SPR events from which the app computes the moving-subtree recurrence table, including per-taxon and per-subtree counts.} Summarising our analysis shows that Phylo-Movies enables direct observation of recombination-associated changes in tree topology: as the sliding window crosses the ORF1/ORF2 junction, \textcolor{red}{polymerase-genotype-associated tree regions dissolve and capsid-genotype-associated tree regions assemble}. The alignment viewer further supports interpretation by reordering sequences according to the current tree layout and colour-coding nucleotide columns, allowing \textcolor{red}{tree-placement patterns} to be inspected alongside matching and mismatching regions in the alignment.

\subsection{Visual Exploration of Candidate Rogue Taxa in Bootstrap Tree Sets}
Rogue taxa are leaves that cannot be reliably placed due to conflicting or missing data, causing them to shift position frequently across a set of reconstructed trees \citep{Wilkinson1996}. Their presence reduces resolution and support in consensus trees; removing them often yields more informative results. Identifying rogue taxa before final analysis is essential.

When analysing bootstrap trees, rogue taxa are those that change position across replicates, indicating unstable phylogenetic placement. \rev{R2c6}{Phylo-Movies is not intended to replace quantitative rogue-taxon detection methods such as RogueNaRok. Instead, it provides a complementary visual layer for inspecting how candidate rogue taxa move across bootstrap replicates: whether they jump diffusely across the tree or repeatedly alternate between a small number of competing placements.} To demonstrate this, we used two multiple sequence alignments from the RogueNaRok dataset \citep{Aberer2013RogueNaRok,RogueNaRokDryad2013}: Dataset~24 (24 taxa, 14{,}190 sites), corresponding to the palaeognath mitochondrial alignment analysed by \citet{Phillips2010}, in which \textit{Ostrich} has been identified as a rogue taxon due to unstable placement across bootstrap replicates; and Dataset~125 (125 taxa, 29{,}149 sites), whose taxa are anonymised (Seq1, Seq2, Seq3, etc.) and were contributed for benchmarking without disclosure of species identities, which we analysed to evaluate performance on a dataset larger than Dataset~24.

To order the resulting trees in a \textcolor{red}{reproducible} trajectory that facilitates visual tracking of positional changes, we sorted replicates by their similarity in state counts to the original (non-resampled) alignment. For each replicate $b$, we computed the state-count vector $\mathbf{c}_b = (n_A, n_C, n_G, n_T, n_{\text{\textcolor{red}{ambiguous/gap}}})$ and measured its deviation from the original alignment using the Euclidean distance:
\begin{equation}
d_b = \sqrt{\sum_{i \in \{A,C,G,T,\text{\textcolor{red}{ambiguous/gap}}\}} (c_{b,i} - c_{\text{original},i})^2}
\end{equation}
\noindent where $c_{b,i}$ and $c_{\text{original},i}$ denote the counts of state $i$ in bootstrap replicate $b$ and in the original multiple sequence alignment, respectively. Replicates were sorted by increasing $d_b$. Bootstrap replicates have no intrinsic biological order; any linear arrangement imposes a trajectory for visualisation. This does not alter the inferred trees themselves, but determines which pairs of trees are compared consecutively for the animation. \textcolor{red}{Bootstrap replicate alignments were generated with RAxML \texttt{raxmlHPC -f j} \citep{Stamatakis2014}, and the maximum-likelihood tree for each replicate alignment was inferred using IQ-TREE~3 in default search mode with SH-aLRT branch support (1{,}000 replicates) \citep{Wong2026IQTREE3}. The promoted Phylo-Movies examples use the resulting 200 IQ-TREE trees per dataset, ordered by ascending composition distance, with internal branch labels annotated as split-frequency support across the same 200 inferred bootstrap-replicate trees. The static Phylo-Movies examples were midpoint-rooted before interpolation, and SPR-move recurrence was computed from these midpoint-rooted tree sequences. Thus, internal branch labels and SH-aLRT metadata describe branch support, whereas the recurrence summaries describe how often taxa or subtrees participate in topology-changing SPR moves across the ordered tree sequence.}

We asked whether taxa identified as rogue by RogueNaRok also appeared among highly recurrent moving subtrees when Phylo-Movies animated this trajectory of bootstrap trees \citep{Aberer2013RogueNaRok}. RogueNaRok identifies rogue taxa by optimising the relative bipartition information criterion (RBIC), which measures the overall support across a bootstrap tree set relative to the maximum achievable support. In contrast, Phylo-Movies does not compute a support-based criterion; instead, it decomposes differences between trees into sequential SPR moves and highlights the subtrees that change position across replicates. \rev{R2c7}{The subtree panels and timeline summaries therefore serve as an exploratory uncertainty view: they show which taxa or subtrees repeatedly participate in SPR moves and where those moves occur in the ordered tree sequence, while support-based filtering and rogue-taxon scoring remain tasks for dedicated quantitative methods.}

In the 24-taxon dataset, based on 200 bootstrap trees, RogueNaRok reported an increase in RBIC from 0.828810 to 0.853333 upon removal of \textit{Ostrich}. Correspondingly, the singleton moving subtree containing \textit{Ostrich} was the \textcolor{red}{most recurrent moving subtree in our SPR move recurrence analysis}, \textcolor{red}{ranking 1st out of 27 recurrent moving subtrees (79 SPR moves; 19.60\% of 403 SPR moves; above 96.3\% of recurrent moving subtrees by recurrence count)}. The complete tables are available in our GitHub repository and can also be accessed directly through the example datasets provided within Phylo-Movies.

In the 125-taxon dataset, also comprising 200 bootstrap replicates, RogueNaRok reported an increase in RBIC from 0.958484 to 0.958549 upon removal of \textit{Seq112}. In the Phylo-Movies recurrence summaries, the singleton moving subtree containing \textit{Seq112} \textcolor{red}{ranked 4th out of 44 recurrent moving subtrees, tied in SPR-move count with ranks 2 and 3 (91 SPR moves; 8.47\% of 1{,}074 SPR moves; above 90.9\% of recurrent moving subtrees by recurrence count)}.

Together, these results show that Phylo-Movies provides an interactive visual approach for exploring \textcolor{red}{SPR-move patterns} of candidate rogue taxa. The RogueNaRok-identified taxa were also highly recurrent in the Phylo-Movies SPR summaries, but the two analyses measure different properties: RogueNaRok optimises a support-based RBIC criterion, whereas Phylo-Movies reports repeated topology-changing movements across the ordered tree sequence.

\textbf{Discussion} 

We have shown two use cases where visualising tree sequences is useful to gain insights that would be difficult to obtain by manual inspection. We illustrated this approach by \rev{R2c6}{visualising placement changes of candidate rogue taxa that substantially destabilize phylogenetic reconstruction} and by inspecting a pronounced recombination breakpoint that underlies the differences observed between norovirus genotypes.

Besides these two use cases, Phylo-Movies visualisation can be used in analysing many use cases involving tree sequences. A common application is the study of continuous topological changes along genomic alignments caused by recombination, a topic of particular interest in virology. When performing population simulations with recombination using tools such as the tskit framework \citep{kelleher2016}, Phylo-Movies can be used to visualise the resulting tree sequence and to observe how recombination events alter genealogical structure along the genome. Moreover, if researchers have performed analyses showing that in some region there are changes in the topology or recombination breakpoints, Phylo-Movies can be used to visualise which subtrees or taxa \textcolor{red}{change parent branch in the corresponding tree sequence}. However, there are no restrictions on the origin of the underlying multiple sequence alignment (MSA) from which the trees were inferred (e.g. bacterial or eukaryotic taxa), nor on the process that generated the resulting tree sequence.

Phylo-Movies can also be used to investigate additional aspects of tree variation along the underlying genomic sequence, including changes in phylogenetic resolution, instability of root or outgroup placement, branch-length variation, and other topology shifts not necessarily caused by recombination. Phylo-Movies also accepts multifurcating trees, expanding its range of applications. This allows users to observe how phylogenetic resolution varies across genomic regions---for instance, identifying windows where certain subtrees collapse into multifurcations due to insufficient informative sites, which may guide the choice of window size or indicate regions of genuine evolutionary uncertainty. 

Similarly, one can check whether an outgroup used for rooting the trees remains stable, or whether the root branch ends in multifurcations (indicating loss of phylogenetic signal) or the outgroup \textcolor{red}{changes parent branch across genomic regions (showing positional instability)}. Both outcomes—loss of resolution and positional instability—show that the outgroup does not consistently root the tree across all genomic regions. Finally, Phylo-Movies is also able to visualise the change of branch lengths along the genomes even if the tree topologies remain the same during the whole range. Such changes may be caused by selection effects, which are sometimes misinterpreted as recombination events in purely distance-based analyses \citep[e.g.][]{worobey2002}.

\rev{R2c16,R3c0}{For methodological diagnostics, users may also arrange a small set of trees inferred under explicitly chosen alternative analysis settings and inspect which subtrees are sensitive to those choices; no intrinsic biological or temporal order of substitution models is implied.} When developing or analysing tree search strategies, one can use the visualisation to study the search trajectory through tree space, identifying which subtrees remain stable early versus those that continue rearranging until the final iterations. \rev{R3c5}{Because the animation exposes otherwise abstract tree rearrangements as visible SPR moves, Phylo-Movies can also be used as an educational tool in teaching settings and hands-on workshops.}

\rev{R3c6}{When analysing tree sequences with many taxa, numerous trees, and frequent rearrangements, the distance plot helps locate regions of topological change so that users need not watch the entire animation to find interesting regions. The practical limit is both computational and perceptual. WebGL rendering supports interactive exploration of hundreds of taxa on typical laptops, as illustrated by the 334-taxon Norovirus dataset and the 125-taxon RogueNaRok example, but label readability, screen or projector resolution, and the number of intermediate transitions become limiting factors as datasets grow. For particularly large or complex datasets, users can select a contiguous range of trees, thin the tree sequence to include only every $k$-th tree, hide labels, or focus on moving subtrees and summary panels. Although only \textcolor{red}{17 trees} were inferred in the Norovirus analysis, the numerous and complex rearrangements between consecutive windows required \textcolor{red}{6{,}391 interpolated intermediate trees} for visualisation. In general, increasing numbers of taxa, trees, and topological changes lead to more intermediate transitions because each rearrangement must be decomposed into sequential SPR moves. This scaling behaviour represents a practical limitation: larger tree sequences require the generation and rendering of more intermediate states, which may reduce responsiveness and make individual SPR moves harder to track in highly complex analyses.}

Like FigTree~\citep{Rambaut2006Figtree} and DensiTree~\citep{Bouckaert2010}, Phylo-Movies reads trees in standard Newick format. However, whereas FigTree displays static images of individual trees and DensiTree summarises a set of trees by overlaying them, Phylo-Movies animates the topological changes between consecutive trees. In addition, Phylo-Movies can display two trees side by side, linking corresponding taxa with connecting lines---similar to a tanglegram\citep{Page2002}---so that users can track the origin and destination of \textcolor{red}{moving subtrees} during the animation. In addition to animating topological transitions, Phylo-Movies provides functionality for alignment inspection, sliding-window tree inference, metadata-based colouring, and video export. These features support exploratory analysis and facilitate the interpretation of topological changes within a tree sequence.
