# Publication Data Relationship Map

Audit date: 2026-05-18

This document explains which source data we used, what was generated by
Phylo-Movies or helper workflows, and how the files should be grouped so a
reader can understand the relationships directly.

## Source Paper Data Used

### Aberer/RogueNaRok Paper

Source paper: Aberer, Krompass, and Stamatakis, "Pruning Rogue Taxa Improves
Phylogenetic Accuracy: An Efficient Algorithm and Webservice", Systematic
Biology 62(1), 2013, DOI `10.1093/sysbio/sys078`.

External archive inspected locally:

```text
/Users/berksakalli/Projects/rogue_taxa_analysis/online-data/
```

The archive README states that it contains data material for the Aberer et al.
paper: real datasets with bootstraps and alignments, algorithm runtime outputs,
additional measurements, the 100K-taxa run, and simulations.

What Phylo-Movies currently uses from that paper archive:

| External archive path | Current use in Phylo-Movies | Status |
| --- | --- | --- |
| `data/datasets/alignments/24` | Used as source alignment for the 24-taxon bootstrap demo generation. | External source, not vendored in `publication_data/`. |
| `data/datasets/alignments/125` | Used as source alignment for the 125-taxon bootstrap demo generation. | External source, not vendored in `publication_data/`. |
| `data/datasets/alignments/350` | Referenced by `rogue_taxa/epas1_pipeline/bootstrap_bridge/generate_bootstrap_order.py`. | Script reference only; not currently a publication demo output. |

What we did not use as publication data:

| External archive area | Reason |
| --- | --- |
| `data/datasets/bootstraps/24` and `data/datasets/bootstraps/125` | The paper archive contains 1000 bootstrap trees. Our `bootstrap_example/` files are 200 newly generated FastTree trees from bootstrap alignments, not byte-identical copies of the paper bootstrap tree sets. |
| `data/additionalMeasurements/` | These are Aberer paper benchmarking/RBIC/RogueNaRok outputs, not Phylo-Movies demo inputs. |
| `data/runtimes/` | These are Aberer paper algorithm timing outputs, not Phylo-Movies demo inputs. |
| `data/100K-taxa-run/` | Not used in the current Phylo-Movies publication layer. |
| `data/simulations/` | Not used in the current Phylo-Movies publication layer. |

Derived Phylo-Movies files from the Aberer alignments:

| Current path | Meaning |
| --- | --- |
| `bootstrap_example/24/all_trees_24.nwk` | 200 FastTree trees generated from RAxML bootstrap alignments based on external alignment 24. |
| `bootstrap_example/24/bootstrap_order_24.txt` | Ordering of the 200 generated replicates by nucleotide-composition distance. |
| `bootstrap_example/24/24-spr-event-analytics.csv` | Phylo-Movies SPR event summary for the 24-taxon tree sequence. |
| `bootstrap_example/125/all_trees_125.nwk` | 200 FastTree trees generated from RAxML bootstrap alignments based on external alignment 125. |
| `bootstrap_example/125/bootstrap_order_125.txt` | Ordering of the 200 generated replicates by nucleotide-composition distance. |
| `bootstrap_example/125/125_spr-event-frequencies.csv` | Phylo-Movies event-frequency summary for the 125-taxon tree sequence. |
| `bootstrap_example/run_log.txt` | Run record for the local bootstrap-ordering generation. |

Planned replacement:

`bootstrap_example/IQTREE_REINFERENCE_PLAN.md` defines the new publication plan:
keep the bootstrap replicate alignments explicit for composition-distance
ordering, replace FastTree tree inference with IQ-TREE 2, regenerate ordered
Newick files and SPR analytics, and retain the current FastTree outputs only as
legacy exploratory data if needed.

`bootstrap_example/DISTANCE_METRIC_AUDIT.md` records a distance-metric bug in
the legacy ordering tables: IUPAC ambiguity symbols were omitted from the vector.
The IQ-TREE re-inference should use the corrected full-cell vector
`(A, C, G, T, AmbiguousOrGap)`.

### Norovirus Recombination Sources

The norovirus example is not from the Aberer/RogueNaRok paper. It is a separate
Phylo-Movies recombination use case derived from Nextstrain norovirus metadata
and GenBank sequences.

| Source layer | Current use |
| --- | --- |
| `norovirus/augur_subsampling/01_raw/full_genome_sequences.fasta` | Raw source sequence layer from Nextstrain/GenBank-derived data. |
| `norovirus/augur_subsampling/03_trimmed/subsampled_350_gappyout_final.fasta` | Canonical 334-sequence publication alignment. |
| `norovirus/augur_subsampling/metadata/subsampled_350_metadata.csv` | Matching metadata for the 334 final sequences. |
| `norovirus/recan_recombination_analysis/` | Derived ReCAN validation workspace; currently not publication-clean because query selection is index-based and stale. |

done with compatibility path updates.

## Practical Grouping To Use Now

Until paths are migrated, readers should interpret the current folders as:

| Current path | Reader-facing role | Future home |
| --- | --- | --- |
| `bootstrap_example/` | Rogue-taxon Phylo-Movies demo outputs derived from Aberer alignments 24 and 125. | `rogue_taxa/phylomovies_bootstrap_demo/` |
| `rogue_taxa/selected_external_files/` | Reserved place for individually justified Aberer/RogueNaRok source files. | Keep under `rogue_taxa/source_aberer_roguenarok/selected_external_files/`. |
| `rogue_taxa/epas1_pipeline/bootstrap_bridge/` | Misplaced conceptually; this is Aberer bootstrap-demo bridge code, not EPAS1 biology. | `rogue_taxa/scripts/bootstrap_ordering/`. |
| `rogue_taxa/epas1_pipeline/scripts/` | EPAS1/archaic-introgression exploratory pipeline scripts. | `rogue_taxa/scripts/epas1_pipeline/` or `rogue_taxa/epas1_pipeline/`. |
| `rogue_taxa/epas1_pipeline/legacy/` | Non-publication-safe exploratory scripts. | `rogue_taxa/legacy/epas1_pipeline/`. |
| `norovirus/augur_subsampling/` | Canonical norovirus recombination alignment and metadata workflow. | `norovirus/phylomovies_demo/` plus `norovirus/source_nextstrain_genbank/`. |
| `norovirus/recan_recombination_analysis/` | ReCAN validation workspace, currently stale. | `norovirus/recan_validation/` after query-selection fix and regeneration. |

- `rogue_taxa/PUBLICATION_FILE_GROUPS.md`
- `rogue_taxa/PUBLICATION_DATA_HYGIENE_AUDIT.md`

For a maintainer who wants to clean the repository:

1. Keep external raw data out of Git unless selected and documented.
2. Move `bootstrap_bridge/` out of `epas1_pipeline/`.
3. Fix ReCAN query selection and regenerate ReCAN outputs.
4. Add manifests/checksums for all retained source and derived publication
   files.
5. Only then consider renaming `bootstrap_example/` to
   `rogue_taxa/phylomovies_bootstrap_demo/` and update app/test references.
