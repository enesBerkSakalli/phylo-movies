# Norovirus Source Provenance

Status date: 2026-05-28

## Primary Source

The norovirus publication data are derived from the Nextstrain norovirus
workflow and public GenBank records.

| Field | Value |
| --- | --- |
| Upstream repository | `https://github.com/nextstrain/norovirus` |
| Upstream workflow | Nextstrain norovirus `ingest/` workflow |
| Local source FASTA | `source_preparation/augur_subsampling/01_raw/full_genome_sequences.fasta` |
| Local source metadata | `source_preparation/augur_subsampling/metadata/full_genome_metadata.tsv` |
| Locked accession versions | `source_preparation/augur_subsampling/01_raw/full_genome_accession_versions.txt` |
| Access date recorded for this audit | 2026-05-19 |
| Upstream commit | `bce398d15a14c82a2a8c3574da289205e2c5844f` |

The upstream repository no longer stores `data/sequences.fasta` and
`data/metadata.tsv` as committed source files. Those files are generated by the
Nextstrain ingest workflow from public NCBI/GenBank records. To make that
moving public-data source reproducible, this archive pins both the Nextstrain
workflow commit and the exact accession-version list used for the retained
local source snapshot.

Recreate the raw source snapshot and downstream 334-taxon alignment with:

```bash
PATH="$PWD/.venv-publication/bin:$PATH" \
  ./publication_data/recombination_norovirus/source_preparation/augur_subsampling/scripts/recreate_nextstrain_augur_snapshot.sh
```

The script runs the pinned Nextstrain ingest workflow with
`nextstrain build --ambient`, locks the generated results to
`full_genome_accession_versions.txt`, then uses `augur filter`, MAFFT, and
trimAl to rebuild the Phylo-Movies alignment inputs.

## Derived Publication Inputs

The canonical Phylo-Movies recombination input is:

```text
source_preparation/augur_subsampling/03_trimmed/subsampled_350_gappyout_final.fasta
```

It contains 334 sequences, all length 8058 bp. The matching metadata file is:

```text
source_preparation/augur_subsampling/metadata/subsampled_350_metadata.csv
```

The ReCAN validation subset is derived from that canonical alignment and is
promoted through:

```text
current_results/
```

## Checksums

File-level SHA256 hashes for the source, intermediate, canonical, and promoted
ReCAN files are recorded in:

```text
MANIFEST.sha256
```

## Release Note

The reproducibility boundary is now explicit: the workflow is pinned to
Nextstrain norovirus commit `bce398d15a14c82a2a8c3574da289205e2c5844f`, and the
data snapshot is pinned to the 4565 accession versions in
`full_genome_accession_versions.txt`. This avoids depending on the mutable
current state of NCBI/GenBank while still recreating the source through
Nextstrain and Augur.
