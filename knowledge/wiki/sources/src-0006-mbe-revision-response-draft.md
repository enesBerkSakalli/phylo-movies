# MBE Revision Response Draft

## Summary

`revision/MBE_revision_response_draft.md` captures planned responses to editor
and reviewer feedback for manuscript `MBE-26-0127`, "Phylo-Movies: Animating
Phylogenetic Trees from Sliding-Window Analyses." Source:
`revision/MBE_revision_response_draft.md`.

## Key Claims

- The revision plan emphasizes software accessibility, clearer scope for the
  GitHub Pages deployment, usability around tree scaling and the HUD, and MSA
  synchronization behavior. Source: `revision/MBE_revision_response_draft.md`.
- Rogue-taxon visualization should be framed as complementary exploratory and
  diagnostic context, not as a replacement for quantitative instability
  measures. Source: `revision/MBE_revision_response_draft.md`.
- FastTree should be described as an exploratory default, while externally
  inferred trees from IQ-TREE, RAxML-NG, or other tools remain supported for
  publication-grade analyses. Source: `revision/MBE_revision_response_draft.md`.
- The public GitHub Pages site should be described as documentation-only; full
  processing requires desktop, Docker, or local full-stack setup. Source:
  `revision/MBE_revision_response_draft.md`.
- The response draft identifies rectangular animated layouts, narrated demos,
  and large-dataset guidance as revision or future-work topics. Source:
  `revision/MBE_revision_response_draft.md`.

## Evidence

The source is organized by editor and reviewer comments and gives planned
responses for software distribution, terminology, FastTree, usability, MSA
synchronization, rogue taxa, scalability, and manuscript presentation.
