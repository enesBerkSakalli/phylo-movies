# Documentation

This directory contains public documentation that should stay useful to
contributors and users.

## Layout

- `architecture/` - stable vocabulary, visual contracts, and design notes.
- `development/` - contributor-facing repository and dependency ownership notes.
- `assets/` - images and other static files referenced by public docs.

Historical spikes, one-off implementation plans, and manuscript revision working
files belong in the knowledge wiki or git history, not in the public docs tree.
